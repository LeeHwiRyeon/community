# 커뮤니티 플랫폼 기획 명세서

## 1. 프로젝트 개요
Community Hub는 React + TypeScript 기반 프론트엔드와 Express 모의 API를 사용하는 커뮤니티 플랫폼입니다. 게시물 탐색, 트렌딩 토픽, 중첩 게시판을 지원하며, 로컬 환경에서 백엔드(포트 50000)와 프론트엔드(포트 5000)로 실행됩니다.

## 1.5. 현재 구현된 기능 (2025년 9월 26일 기준)

### ✅ 완료된 핵심 기능
- **투표 시스템**: 게시물에 대한 찬성/반대 투표, 실시간 투표 수 표시
- **태그 시스템**: 게시물 태그 입력, 태그 클라우드, 태그 기반 필터링
- **알림 시스템**: 실시간 알림 센터, 알림 설정 관리
- **사용자 프로필 시스템**: 프로필 카드, 프로필 페이지, 사용자 정보 표시
- **검색 시스템**: 키워드 검색, 검색 결과 페이지, 필터링 옵션
- **댓글 시스템**: 계층적 댓글 구조, 댓글 표시 컴포넌트
- **게시물 조회수 시스템**: 조회수 카운팅, 조회수 표시
- **방송 게시판**: 라이브 방송 표시, 스포일러 토글, 방송 카드 컴포넌트

### 🔧 기술 스택
- **프론트엔드**: React 18, TypeScript, Vite, Chakra UI, TanStack Query, Vitest
- **백엔드**: Express.js, REST API, MySQL 데이터베이스
- **테스트**: Vitest, React Testing Library, @testing-library/jest-dom

### 📊 데이터베이스 스키마
- posts, users, boards, votes, tags, notifications, comments, post_views, broadcasts 테이블
- 외래 키 제약조건 및 인덱스 최적화

#### RPG 스타일 프로필 시스템 (게이미피케이션)
**현재 트렌드 평가**: 옛날 인벤/PlayXP 스타일의 RPG 요소는 요즘에도 게임 커뮤니티나 포럼에서 인기 있지만, 간소화된 형태로 진화했습니다. Reddit의 카마나 Stack Overflow의 배지 시스템처럼 가볍게 적용됩니다.

**제안 기능**:
- **레벨 시스템**: 게시물/댓글 수에 따른 레벨업 (예: Lv.1 초보자 → Lv.10 전문가)
- **배지/칭호**: 활동 기반 획득 (첫 게시물, 100개 댓글 등)
- **스탯 포인트**: 활동성, 도움됨 점수, 커뮤니티 기여도 표시
- **아바타 커스터마이징**: 기본 아바타 선택 또는 간단한 커스터마이징
- **프로필 카드**: RPG 스타일 카드 형태로 프로필 표시

**장점**: 사용자 참여 유도, 재미 요소, 커뮤니티 로열티 증가
**단점**: 과도한 게이미피케이션은 부담, 유지보수 복잡
**추천**: 최소 버전으로 시작 (레벨 + 배지만), 사용자 피드백 후 확장

## 3. 커뮤니티 기능 분석 및 개선 제안

### 주요 커뮤니티 플랫폼 비교 및 핵심 기능 검토

#### Reddit (대규모 토론 중심)
**핵심 기능**:
- **투표 시스템**: 게시물/댓글에 대한 업보트/다운 보트로 콘텐츠 순위 결정
- **서브레딧**: 주제별 커뮤니티 분리 및 독립적 규칙 운영
- **모더레이터 도구**: 게시물 삭제, 사용자 밴, 규칙 적용 자동화
- **실시간 토론**: 댓글 스레드 및 라이브 토론 기능
- **알고리즘 피드**: 사용자 관심사 기반 콘텐츠 추천

**강점**: 대규모 커뮤니티 관리 효율성, 알고리즘 기반 발견성
**약점**: 과도한 익명성으로 인한 트롤링, 콘텐츠 품질 저하 가능성
**적용 가능 기능**:
- 투표 시스템 (업보트/다운 보트)
- 게시판별 규칙 및 모더레이터 권한
- 콘텐츠 추천 알고리즘

#### Discord (실시간 커뮤니케이션 중심)
**핵심 기능**:
- **실시간 채팅**: 텍스트/음성 채널 기반 즉시 커뮤니케이션
- **역할 기반 권한**: 관리자, 모더레이터, 일반 사용자 역할 구분
- **봇 통합**: 자동화된 작업 및 커뮤니티 관리 지원
- **이벤트 시스템**: 서버 이벤트 및 일정 관리
- **스레드 및 포럼 채널**: 주제별 토론 분리

**강점**: 실시간 상호작용, 커뮤니티 이벤트 용이성
**약점**: 채팅 기반으로 장기 콘텐츠 보존 어려움, 노이즈 발생 가능성
**적용 가능 기능**:
- 실시간 댓글 및 채팅 시스템
- 역할 기반 권한 관리
- 이벤트 알림 및 일정 관리

#### Stack Overflow (전문 Q&A 중심)
**핵심 기능**:
- **질문-답변 형식**: 구조화된 Q&A 포맷
- **투표 기반 답변 선정**: 커뮤니티 투표로 베스트 답변 결정
- **태그 시스템**: 기술/주제별 분류 및 전문성 표시
- **평판 시스템**: 활동 기반 신뢰도 점수
- **코드 구문 강조**: 개발 관련 콘텐츠 최적화

**강점**: 전문 지식 공유, 콘텐츠 품질 관리
**약점**: 일반 토론 부적합, 엄격한 포맷으로 진입장벽 존재
**적용 가능 기능**:
- Q&A 형식 게시판 옵션
- 태그 기반 검색 및 필터링
- 전문성 기반 신뢰도 시스템

#### 기타 플랫폼 참고 (Twitter/X, Facebook Groups)
- **실시간 피드**: 팔로우 기반 콘텐츠 스트림

# 버그 트레커 우선도 및 규칙

## 버그 우선도 레벨
- **Critical (P0)**: 시스템 다운, 데이터 손실, 보안 취약점, 주요 기능 불가
- **High (P1)**: 주요 기능 장애, 사용자 경험 심각 저하, 성능 이슈
- **Medium (P2)**: 부 기능 장애, UI/UX 불편, 성능 저하
- **Low (P3)**: 사소한 버그, 개선 사항, 미래 기능

## 버그 등록 규칙
1. **즉시 등록**: 버그 발견 시 바로 TODO 리스트에 추가 (manage_todo_list 도구 사용)
2. **명세서 작성**: 버그 설명, 재현 방법, 영향 범위, 우선도 포함
3. **증거 수집**: 스크린샷, 로그, 에러 메시지 첨부
4. **할당 및 추적**: 담당자 할당, 진행 상태 업데이트
5. **해결 후 검증**: 수정 후 테스트 및 재발 방지 확인

## 버그 트레킹 프로세스
- 발견 → 등록 (TODO) → 분석 → 수정 → 테스트 → 완료
- 주기적 리뷰: 매일 TODO 리스트 검토 및 우선순위 조정

## 분석 개요

### 분석 대상 플랫폼
본 보고서는 현재 가장 성공적인 커뮤니티 플랫폼들의 디자인 패턴을 분석하여, 우리 Community Hub 플랫폼의 UI/UX 개선 방향을 제시합니다.

**분석 대상**:
- **Reddit**: 투표 기반 콘텐츠 큐레이션 (DAU 5천만+)
- **Discord**: 실시간 커뮤니케이션 (DAU 1.5억+)
- **Stack Overflow**: 전문 지식 공유 (월간 방문자 1억+)
- **GitHub**: 개발자 협업 플랫폼 (월간 활성 사용자 9천만+)

### 분석 방법론
1. **사용자 플로우 분석**: 주요 사용자 시나리오별 인터랙션 패턴
2. **시각적 계층 구조 분석**: 정보 우선순위 및 레이아웃 패턴
3. **상호작용 패턴 분석**: 클릭, 스크롤, 제스처 기반 행동 유도
4. **모바일/데스크톱 적응성 분석**: 반응형 디자인 전략

## 플랫폼별 디자인 패턴 분석

### 1. Reddit: 투표 중심 콘텐츠 큐레이션

#### 핵심 디자인 패턴
**투표 시스템의 시각적 위계**:
- 투표 버튼을 콘텐츠 카드 최상단에 배치 (녹색/빨강 화살표)
- 실시간 점수 표시 및 색상 코딩 (상승/하락 추세)
- 투표 애니메이션으로 즉각적 피드백 제공

**콘텐츠 소비 플로우**:
- 무한 스크롤 기반 피드
- 카드 기반 레이아웃으로 빠른 스캔 가능
- 썸네일 우선 로딩으로 인지적 부하 감소

#### 성공 요인
- **알고리즘적 큐레이션**: 사용자 투표가 콘텐츠 순위 결정
- **커뮤니티 자율성**: 서브레딧별 독립적 규칙과 디자인
- **심플한 인터랙션**: 최소한의 클릭으로 투표/댓글/공유 가능

#### 우리 플랫폼 적용 인사이트
- 투표 버튼 위치 최적화 (현재 위치에서 상단 이동 고려)
- 점수 변화 애니메이션 추가
- 서브커뮤니티별 커스터마이징 옵션 제공

### 2. Discord: 실시간 협업 인터페이스

#### 핵심 디자인 패턴
**채널 기반 네비게이션**:
- 좌측 사이드바: 서버/채널 계층 구조
- 우측 사이드바: 온라인 멤버 및 프로필 정보
- 중앙 메인: 실시간 메시지 플로우

**실시간성 강조**:
- 새 메시지 배지 및 알림
- 타이핑 인디케이터 ("누군가 입력 중...")
- 온라인 상태 표시 (녹색/노랑/회색)

#### 성공 요인
- **즉각적 커뮤니케이션**: WebSocket 기반 실시간 업데이트
- **맥락 유지**: 채널별 대화 맥락 보존
- **시각적 피드백**: 모든 액션에 즉각적 응답

#### 우리 플랫폼 적용 인사이트
- 실시간 알림 배지 시스템 도입
- "최근 활동" 표시기 추가
- 읽음/안읽음 상태 시각화

### 3. Stack Overflow: 전문 지식 계층화

#### 핵심 디자인 패턴
**Q&A 구조 최적화**:
- 질문/답변 명확한 구분
- 베스트 답변 별도 하이라이트
- 평판 점수 기반 신뢰도 표시

**콘텐츠 품질 관리**:
- 태그 기반 필터링 UI
- 투표 기반 답변 순위
- 코드 구문 강조 일관성

#### 성공 요인
- **품질 중심**: 투표와 편집으로 콘텐츠 품질 유지
- **전문성 강조**: 평판 시스템으로 전문가 참여 유도
- **검색 최적화**: 태그와 키워드로 정확한 검색 지원

#### 우리 플랫폼 적용 인사이트
- 베스트 댓글/답변 하이라이트 기능
- 태그 기반 필터링 UI 개선
- 사용자 평판/신뢰도 표시 시스템

### 4. GitHub: 협업 중심 인터페이스

#### 핵심 디자인 패턴
**이슈/토론 기반 구조**:
- 이슈: 문제 제기 및 추적
- 풀 리퀘스트: 코드 리뷰 및 협업
- 토론: 아이디어 공유

**협업 기능 강조**:
- 멘션 (@username) 자동완성
- 리액션 이모지 (👍 👎 ❤️)
- 스레드 답글 구조

#### 성공 요인
- **투명한 협업**: 모든 변경사항 추적 가능
- **비동기 커뮤니케이션**: 시간대 차이 극복
- **맥락 보존**: 코드와 논의 연결

#### 우리 플랫폼 적용 인사이트
- 댓글에 리액션 이모지 추가
- 멘션 자동완성 기능
- 토론 스레드 구조 도입

## 디자인 시스템 구축 전략

### 1. 컴포넌트 라이브러리 설계
**기초 컴포넌트**:
- Button: Primary/Secondary/Ghost variants, 로딩 상태, 아이콘 지원
- Card: 그림자 깊이별 variants, 호버 효과
- Input: 유효성 검사 표시, 에러 상태
- Modal/Dialog: 오버레이, ESC 키 지원, 포커스 트랩

**복합 컴포넌트**:
- PostCard: 투표 + 콘텐츠 + 액션 버튼 통합
- CommentThread: 계층적 댓글 구조
- NotificationToast: 실시간 알림 표시

### 2. 디자인 토큰 시스템
**색상 토큰**:
- Primary: #0066CC (신뢰성, 주요 액션)
- Success: #28A745 (긍정적 피드백)
- Warning: #FFC107 (주의 요구)
- Error: #DC3545 (에러 표시)

**타이포그래피 스케일**:
- Heading 1: 2.25rem (36px) - 페이지 제목
- Heading 2: 1.875rem (30px) - 섹션 제목
- Body Large: 1.125rem (18px) - 주요 콘텐츠
- Body: 1rem (16px) - 일반 텍스트
- Caption: 0.875rem (14px) - 보조 정보

### 3. 반응형 브레이크포인트
- Mobile: < 768px
- Tablet: 768px - 1024px
- Desktop: > 1024px

**모바일 퍼스트 접근**:
- 터치 타겟 최소 44px
- 스와이프 제스처 지원
- 하단 네비게이션 우선

## 스트리머 온보딩 매뉴얼

### 개요
스트리머 온보딩 매뉴얼은 완전 초보자를 대상으로 한 종합 가이드로, 6단계(준비/설정/테스트/커뮤니티/수익화/고급)로 구성됩니다. 목표는 첫 방송 완료율 80% 달성입니다.

### OBS 설정 가이드

#### 1. OBS 다운로드 및 설치
OBS Studio는 무료 오픈소스 방송 소프트웨어입니다.

**단계별 설치 과정:**
1. OBS 공식 웹사이트( https://obsproject.com ) 방문
2. "Download" 버튼 클릭
3. 운영체제에 맞는 버전 선택 (Windows/macOS/Linux)
4. 설치 파일 다운로드 및 실행
5. 설치 마법사 따라 진행 (기본 설정 권장)
6. 설치 완료 후 OBS 실행

**팁:** 최신 버전을 사용하세요. 자동 업데이트를 활성화하세요.

#### 2. 기본 인터페이스 설명
OBS를 처음 실행하면 다음과 같은 인터페이스가 나타납니다:

- **Scenes (씬)**: 방송 화면 레이아웃 관리
- **Sources (소스)**: 화면에 표시할 요소들 (화면 캡처, 웹캠 등)
- **Audio Mixer**: 오디오 입력/출력 제어
- **Preview**: 실제 방송 화면 미리보기
- **Controls**: 시작/중지 버튼과 설정

#### 3. 씬/소스 개념 설명
- **씬(Scene)**: 방송 화면의 템플릿. 게임 방송, 채팅 방송 등 다른 레이아웃
- **소스(Source)**: 씬에 추가하는 요소. 이미지, 텍스트, 화면 캡처, 웹캠 등

**기본 씬 생성:**
1. Scenes 박스에서 "+" 클릭
2. 씬 이름 입력 (예: "메인 방송")
3. Sources에서 요소들 추가

#### 4. 오디오 설정 (마이크/데스크톱)
**마이크 설정:**
1. Audio Mixer에서 Mic/Aux 입력 선택
2. 설정 아이콘 클릭 → Properties
3. 마이크 장치 선택
4. 볼륨 레벨 조정

**데스크톱 오디오:**
1. Desktop Audio 선택
2. 시스템 사운드 출력 장치 선택
3. 필요시 필터 추가 (노이즈 게이트, 컴프레서)

#### 5. 비디오 설정 (웹캠/화면 캡처)
**웹캠 추가:**
1. Sources에서 "+" → Video Capture Device
2. 장치 선택 (웹캠)
3. 해상도 및 프레임레이트 설정

**화면 캡처:**
1. Sources에서 "+" → Display Capture (또는 Window Capture)
2. 캡처할 화면/창 선택
3. 크기 및 위치 조정

#### 6. 인코딩 설정 (해상도/비트레이트/프레임레이트)
Settings → Output → Streaming 탭:

- **비디오 비트레이트**: 3000-6000 Kbps (인터넷 속도에 따라)
- **오디오 비트레이트**: 128-320 Kbps
- **키프레임 간격**: 2초
- **프리셋**: veryfast 또는 faster
- **프로필**: main

**권장 설정 (1080p 60fps):**
- 해상도: 1920x1080
- 프레임레이트: 60
- 비트레이트: 4500 Kbps

#### 7. 스트리밍 설정 (서버/스트림키)
Settings → Stream 탭:

1. Service: Twitch/YouTube 등 플랫폼 선택
2. Server: 가장 가까운 서버 선택
3. Stream Key: 플랫폼에서 복사한 스트림키 입력 (보안 주의!)

**스트림키 보안:**
- 절대 공유하지 마세요
- 정기적으로 변경하세요
- 2FA 활성화

#### 8. 녹화 설정
Settings → Output → Recording 탭:

- **형식**: MP4 또는 FLV
- **비디오 인코더**: x264
- **오디오 트랙**: 1,2 (마이크+데스크톱)
- **저장 경로**: 편리한 폴더 선택

#### 9. 단축키 설정
Settings → Hotkeys 탭:

**필수 단축키:**
- Start Streaming: F8
- Stop Streaming: F9
- Start Recording: F10
- Stop Recording: F11
- Switch Scene: F1-F4 (씬별)

**사용 팁:** 방송 중 실수 방지를 위해 직관적인 키 설정

### 플랫폼 계정 연동 가이드

#### Twitch 계정 생성 및 이메일 인증
1. twitch.tv 방문
2. "Sign Up" 클릭
3. 사용자명, 비밀번호, 생년월일 입력
4. 이메일 주소 입력 및 인증 메일 확인
5. 전화번호 인증 (선택사항이나 보안 권장)
6. 프로필 설정 완료

**사용자명 팁:** 영문+숫자 조합, 기억하기 쉬운 이름 선택

#### 채널 설정 (프로필/배너/설명)
**프로필 설정:**
1. 채널 페이지 우상단 프로필 사진 클릭
2. "Channel" → "Profile Settings"
3. 프로필 사진 업로드 (400x400px 권장)
4. 채널 설명 작성 (자기소개, 방송 스타일)
5. 소셜 링크 추가 (Twitter, Instagram 등)

**배너 설정:**
1. "Channel" → "Customize Channel"
2. 배너 이미지 업로드 (1200x400px)
3. 채널 색상 테마 선택

#### 스트림키 확인 및 보안
1. 채널 페이지 → 설정 아이콘 → "Settings"
2. "Stream Key" 탭 클릭
3. "Show Key" 버튼으로 스트림키 확인
4. OBS 등 소프트웨어에 입력

**보안 팁:**
- 스트림키 절대 공유 금지
- 정기적으로 키 재생성
- 2단계 인증 활성화

#### YouTube Live 설정
1. youtube.com 방문 및 로그인
2. 우상단 프로필 → "YouTube Studio"

#### 장비 점검 체크리스트
**하드웨어 점검:**
- [ ] 컴퓨터 사양 확인 (CPU, RAM, GPU)
- [ ] 인터넷 속도 테스트 (업로드 5Mbps 이상)
- [ ] 마이크 작동 확인 (잡음 없음)
- [ ] 웹캠 화질 테스트

**소프트웨어 점검:**

- [ ] 테스트 스트리밍 실행
- [ ] 녹화 기능 테스트

#### 네트워크 속도 테스트
**테스트 방법:**
1. speedtest.net 방문
2. "Go" 버튼 클릭
3. 다운로드/업로드 속도 확인

**권장 속도:**
- 최소: 다운로드 10Mbps, 업로드 5Mbps
- 권장: 다운로드 50Mbps, 업로드 10Mbps
- 4K 방송: 업로드 20Mbps 이상

**네트워크 최적화:**
- 이더넷 케이블 사용 (Wi-Fi보다 안정적)
- 다른 기기 대역폭 사용 제한
- QoS 설정으로 스트리밍 우선순위 부여

#### 마이크 테스트
**테스트 절차:**
1. OBS Audio Mixer에서 마이크 선택
2. "Configure Audio" → "Properties"
3. 테스트 녹음 실행
4. 녹음 파일 재생하여 품질 확인

**품질 개선 팁:**

### 문제 해결 및 FAQ

#### 네트워크 문제 해결 (방화벽/포트포워딩)
**방화벽 설정:**
1. Windows Defender 방화벽 → 고급 설정
2. 인바운드 규칙 → OBS 허용
3. 포트 1935 (RTMP) 개방
4. VPN/프록시 비활성화

**포트 포워딩 (라우터):**
1. 라우터 관리 페이지 접속 (192.168.1.1)
2. 포트 포워딩 설정
3. 내부 IP 주소 입력
4. 외부 포트: 1935, 내부 포트: 1935
5. 프로토콜: TCP/UDP

**네트워크 최적화:**
- QoS 설정으로 스트리밍 우선순위
- 배경 다운로드 중지
- 이더넷 케이블 사용

#### 오디오 문제 (에코/잡음 제거)
**에코 제거:**
1. OBS Audio Mixer → 마이크 설정
2. Filters → Noise Suppression 추가
3. Method: RNNoise 선택
4. 팝 필터 사용 (마이크 앞)

**잡음 제거:**
- Noise Gate 필터 추가
- Close Threshold: -32dB
- Open Threshold: -26dB
- Attack: 10ms

**하드웨어 솔루션:**
- USB 마이크 사용 (내장 마이크 피하기)
- 오디오 인터페이스 고려
- 조용한 환경 확보

#### 비디오 문제 (랙/저화질)
**랙 현상 해결:**
1. 인코딩 설정 낮추기
2. 프레임레이트 30fps로 변경
3. 해상도 720p로 낮추기
4. x264 프리셋: veryfast

**저화질 문제:**
- 비트레이트 증가 (최대 6000 Kbps)
- 키프레임 간격 2초로 설정
- CPU 사용량 모니터링
- GPU 인코딩 활성화 (가능한 경우)

**화면 끊김:**
- V-sync 비활성화
- 모니터 리프레시 레이트 확인
- 게임 최적화 설정

#### 플랫폼 연결 실패

#### 자주 묻는 질문 (FAQ)

**Q: 방송이 버벅거려요**
A: 인터넷 속도를 확인하세요. 업로드 속도가 5Mbps 이상 필요합니다. OBS 설정에서 비트레이트를 낮추거나 해상도를 줄여보세요.

**Q: 소리가 안 들려요**
A: OBS Audio Mixer에서 데스크톱 오디오가 활성화되어 있는지 확인하세요. 볼륨 레벨을 확인하고, 시스템 믹서를 체크하세요.

**Q: Twitch에 연결이 안 돼요**
A: 스트림키가 정확한지 확인하세요. 방화벽이 OBS를 차단하지 않는지 체크하고, 다른 서버를 시도해보세요.

**Q: 녹화 파일이 커요**
A: MP4 대신 FLV 포맷을 사용하거나, 비트레이트를 낮추세요. 필요 없는 부분은 편집 소프트웨어로 제거하세요.

**Q: 시청자가 저화질로 봐요**
A: Twitch 대시보드에서 스트림 품질을 확인하세요. 인코딩 설정을 조정하고, 인터넷 연결을 개선하세요.

**Q: 채팅이 안 보여요**
A: OBS에 브라우저 소스로 채팅을 추가하세요. Twitch 채팅 URL을 입력하고, 투명 배경으로 설정하세요.

### 비주얼 가이드 및 리소스

#### OBS 인터페이스 라벨링
**OBS 메인 인터페이스 구성:**
```
┌─────────────────────────────────────┐
│ Scenes     Sources     Controls     │ ← 상단 메뉴
├─────────────────┬───────────────────┤
│                 │                   │
│   Scenes        │   Preview         │ ← 좌측: 씬 목록
│   (방송 레이아웃) │   (미리보기)      │    우측: 방송 화면
│                 │                   │
├─────────────────┴───────────────────┤
│ Audio Mixer (오디오 믹서)           │ ← 하단: 음성 제어
└─────────────────────────────────────┘
```

**주요 버튼 설명:**
- **Start Streaming**: 방송 시작 (빨강 버튼)
- **Stop Streaming**: 방송 중지
- **Start Recording**: 녹화 시작
- **Settings**: 환경설정 (톱니바퀴 아이콘)

#### 플랫폼 설정 화면 캡처

**타겟 사용자:** 방송 경험이 없는 초보자

**형식:** 텍스트 가이드 + 비디오 튜토리얼 + 대화형 웹사이트
